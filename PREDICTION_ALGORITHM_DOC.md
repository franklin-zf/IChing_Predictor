# 预测算法架构与功能演进文档 (Prediction Algorithm Documentation)

## 📌 1. 算法体系概述 (Overview)
本项目旨在为大盘指数（QQQ, SPY等）提供日内高频的**方向性前瞻**与**关键点位图谱**。
算法体系已经历了从纯空间矩阵（V1）到“量价时空+易经空间”（V2）的跨越。本规范要求：**任何未来对预测算法的变更、修改与特征扩展，必须优先在此文档中完成理论推理与登记，随后再进行代码落地。**

---

## ☯️ 2. V1 核心预测逻辑 (I-Ching 纯空间矩阵)
V1 版本的核心基于**中国易经模型**与**现代量化指标**的融合。
* **基本指标与中枢 (Pivot)**：采用前一日的 VWAP（成交量加权平均价）与高、低、收盘价加权，构成日内多空分水岭（Pivot）。
* **波动率基准 (ATR)**：基于过去 5 日的真实波动幅度（ATR_5），并结合不同宏观事件（如 CPI、FOMC）赋予不同的“爆发乘数 (Multiplier)”，确定今日的最大震幅空间。
* **起卦与排布**：
  - 上卦：前一日（High + Low）映射。
  - 下卦：当前开盘价（今日 Open + 昨日 Close）映射。
  - 动爻：结合日期与宏观因子的时间变量。
* **六大极值空间位**：结合**缠论顶底分型**作为极端保护位，以 Fibonacci 比例向上向下延展，将卦象阴阳赋能于 6 个具体价格上，形成日内阻力与支撑。

## 📊 3. V2 优化逻辑：第一性原理量价时空映射 (Quant Volume-Gap Analysis)
基于2024-2026年超1000个高频交易日的深度大数据回测，V1的空间矩阵（点位）非常精准，但在**“何时进场”、“做左侧还是右侧”**上缺乏宏观定势。
因此，V2 引入了**盘前跳空幅度（Gap %）**与**夜盘/盘前相对成交量（Volume Ratio）**作为前置过滤器。

### 3.1 核心驱动指标 (New Features)
1. **盘前跳空幅度 (Gap %)**：`(今日盘前最新价 - 昨日收盘价) / 昨日收盘价 * 100`
2. **夜盘相对成交量倍数 (Overnight Volume Ratio)**：`今日夜盘成交量 / 过去20日平均夜盘成交量`

### 3.2 量价定势规则引擎 (Bias Engine)
* **状态 A：震荡均值回归 (Mean Reversion)**
  * **触发条件**：`0.5% <= abs(Gap) < 1.5%` 且 `Volume Ratio < 2.0`。
  * **实战指导**：高开低走 / 低开高走概率极大（~65%胜率）。大盘动能透支，需左侧逆势操作（高空/低买）。
  * **时空窗口**：逢高做空窗口在 **09:30 - 09:45**（诱多极值）；逢低抄底窗口在 **09:40 - 10:15**（恐慌竭尽）。
* **状态 B：单边动能延续 (Trending / Squeeze)**
  * **触发条件**：极端跳空 `abs(Gap) >= 1.5%` **或** 盘前出现异动巨量换手 `Volume Ratio >= 2.0`。
  * **实战指导**：均值回归失效。基本面重构，盘前认输盘出清，开盘切忌逆势。开盘后顺着跳空方向右侧做单。
* **状态 C：常态震荡 (Neutral)**
  * **触发条件**：`abs(Gap) < 0.5%`。
  * **实战指导**：无明显开盘抛压，完全依赖 V1 的易经 Pivot 中枢与 6 个动爻点位进行高抛低吸。

## ⚙️ 4. V2 算法执行流水线 (Algorithm Flow)
1. **数据拉取与清洗**：严格分离 16:00-09:30 的夜盘/盘前数据与 09:30-16:00 的 RTH 正盘数据。
2. **动态资产折算 (Price Scaling)**：由于历史数据库存储的可能是标的资产的指数数据（如 `SPX`，指数级 6600+），而实时盘前拉取的是衍生 ETF 数据（如 `SPY`，ETF级 660+）。在接入实时盘前快照时，必须根据昨日收盘价计算出折算比例（Scale Ratio），将实时盘前价格放大还原至与历史数据一致的量级。**绝对禁止直接将两套不同标度的数据进行数学运算，否则会导致跳空缺口和中枢发生致命偏离**。
3. **计算指标**：计算过去 5 日的 ATR、20日的日内/夜盘平均成交量。
4. **提取盘前快照**：获取截至目前（或 09:30）的最新盘前价格（已折算）和夜盘累计成交量。
5. **量价定势诊断**：通过 3.2 的规则引擎，输出今日的宏观 Bias（Mean_Reversion / Trending / Neutral）与**最佳出击时间窗口**。
6. **点位矩阵计算**：使用 V1 逻辑，计算 Pivot、爆发 ATR 以及第 1 到 6 爻的空间点位。
7. **聚合输出**：生成最终的预测战报 Markdown。

## 📝 5. 错误边界与数据对齐约束 (Data Alignment Constraints)
在后续的功能迭代中，必须严格遵循以下数据对齐约束：
* **标度对齐 (Scale Alignment)**：引入任何新的数据源或关联资产前，必须显式核对价格标度。例如在预测 SPY 期权时，不能直接将 SPY 的 660 价格带入到 SPX 的 6600 算法中，需要通过 `Scale Ratio = SPX_Close / SPY_Close` 动态换算。
* **数据空值回退 (NaN Fallback)**：当节假日或上游数据源丢失导致某一日 `High/Low/Close` 为 NaN 时，算法不得抛出错误阻断执行，应触发 Fallback 机制向前追溯最近一个有效交易日，防止易经起卦时发生 Int 类型转换崩溃。

## 📝 6. 版本变更控制 (Version Control Policy)
* **任何阈值修改**（如将极端跳空从 1.5% 改为 1.2%），必须先通过全量高频数据的回测脚本验证，胜率显著上升后，方可更新本文档，最后落入 Python 代码。
* **新增因子**（如加入 VIX 恐慌指数或期权 Gamma 暴露），需在文档中新增第 3.3 章节阐述第一性原理逻辑，并定义计算口径。
